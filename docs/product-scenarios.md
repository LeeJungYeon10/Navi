# 안녕 나비야 사용자 시나리오

## 제품 포지션

안녕 나비야는 상담 앱보다 가벼운 정서 기록 웹앱이다. 사용자는 긴 일기를 쓰거나 분석을 받기보다, 화면 안에서 돌아다니는 나비에게 오늘 상태를 짧게 말하고 하루의 발자국을 남긴다.

첫 타깃은 10~30대다. 이 타깃에게 중요한 경험은 부담 없는 진입, 귀여운 상호작용, 짧은 기록, 다시 열어보고 싶은 정서적 흔적이다.

## 나비의 역할과 말투

나비는 반려묘형 동반자다. 판단하거나 진단하지 않고, 사용자의 말을 짧게 받아주고 오늘 상태를 함께 정리한다.

- 말투: 다정하지만 과하게 상담사처럼 굴지 않음
- 역할: 감정 요약, 작은 루틴 제안, 재방문 동기 제공
- 피해야 할 것: 진단, 치료 표현, 과한 위로, 긴 설교
- 지향점: "나비한테 한마디 남겼다"는 가벼운 감각

## 핵심 사용자 시나리오

사용자는 하루 중 지치거나 기분이 복잡하거나, 누군가에게 한마디 하고 싶을 때 앱을 연다. 나비가 화면 안에서 다가오고, 사용자는 길게 쓰지 않아도 된다.

예를 들어 "오늘 좀 피곤해", "마음이 불안해", "그냥 별일 없었어" 정도만 남긴다. 나비는 그 말을 바탕으로 짧게 공감하고, 오늘의 상태를 요약한다. 마지막에 사용자는 오늘의 발자국을 저장한다.

## MVP 핵심 기능

1. 나비와의 짧은 대화
2. 오늘의 발자국 저장
3. 최근 발자국 보기

첫 MVP는 기능을 많이 늘리기보다 "말하기 -> 나비 피드백 -> 요약 저장 -> 다시 보기" 흐름을 선명하게 만드는 데 집중한다.

## 데이터 저장 원칙

MVP에서는 대화 원문을 서버에 저장하지 않는다. 사용자가 확인한 오늘의 발자국 요약만 저장한다.

저장 범위:

- 감정 상태: `mood`
- 수면 상태: `sleep`
- 활동 상태: `activity`
- 나비의 한 줄 요약: `nabi_note`
- 사용자 선택 메모: `user_note`
- 추천 루틴: `routine`
- 루틴 완료 여부: `routine_done`

이 원칙은 개인정보 부담을 낮추고, 웹앱 링크 배포 단계에서 신뢰를 만들기 위한 기준이다.

## 첫 방문 흐름

1. 사용자가 링크를 눌러 앱에 들어온다.
2. 랜딩 페이지 없이 바로 나비와 대화 화면을 본다.
3. 나비가 짧게 인사한다.
4. 사용자는 닉네임과 관심 목표를 최소한으로 설정한다.
5. 첫 대화를 시작한다.
6. 대화가 끝나면 첫 발자국을 저장한다.

## 매일 사용 흐름

1. 사용자가 앱을 연다.
2. 나비가 오늘 상태를 묻는다.
3. 사용자는 버튼 또는 자유 입력으로 답한다.
4. 나비가 짧은 피드백을 준다.
5. 오늘의 발자국 저장 여부를 묻는다.
6. 저장 후 작은 루틴 하나를 제안한다.

## 재방문 흐름

1. 사용자가 발자국 탭을 연다.
2. 최근 3~7일의 감정, 수면, 활동 흐름을 본다.
3. 나비가 "요즘 피곤한 날이 조금 많았어"처럼 가볍게 패턴을 말한다.
4. 사용자는 오늘 할 수 있는 작은 행동을 선택한다.
5. 다시 대화로 돌아온다.

## 대표 사용자

### 10대 후반~20대 초반

일기 앱은 부담스럽지만 귀여운 캐릭터에게 말하는 건 괜찮은 사용자다. 핵심 가치는 감정 기록보다 나비와 대화한 흔적이다.

### 20대 대학생

과제, 인간관계, 생활 리듬이 흔들릴 때 사용한다. 핵심 가치는 부담 없는 감정 정리와 짧은 위로다.

### 20~30대 직장인

퇴근 후 피로와 무기력감을 정리하고 싶을 때 사용한다. 핵심 가치는 오늘 상태 요약과 내일을 위한 작은 회복 루틴이다.

## 첫 7일 사용자 여정

### 1일차: 첫 만남

목표는 앱의 부담을 낮추는 것이다. 나비가 사용자를 반기고, 첫 발자국을 쉽게 남기게 한다.

### 2일차: 다시 말 걸기

목표는 재방문 장벽을 낮추는 것이다. 어제 남긴 발자국을 짧게 언급하고 오늘 상태를 묻는다.

### 3일차: 기록 습관 만들기

목표는 사용자가 기록 방식을 이해하는 것이다. 짧은 입력만으로도 발자국이 만들어진다는 점을 보여준다.

### 4일차: 작은 패턴 보여주기

목표는 기록의 의미를 느끼게 하는 것이다. 최근 감정이나 수면 흐름을 과장 없이 요약한다.

### 5일차: 루틴 제안

목표는 행동으로 이어지는 작은 도움을 주는 것이다. 호흡, 물 마시기, 산책, 스트레칭 같은 낮은 부담의 루틴을 제안한다.

### 6일차: 나비와의 관계감

목표는 도구보다 동반자처럼 느끼게 하는 것이다. 나비가 사용자의 반복 패턴을 기억하는 듯한 피드백을 준다.

### 7일차: 이번 주의 발자국

목표는 계속 쓸 이유를 만드는 것이다. 일주일의 발자국을 요약하고 다음 주에 이어갈 작은 목표를 제안한다.

## 출시 방향

초기 출시는 앱스토어보다 웹앱 링크 배포로 간다. 링크 기반 배포가 빠르고, 10~30대 타깃에게 공유와 테스트가 쉽다.

추천 순서:

1. GitHub 저장소 공개
2. 웹앱 배포 URL 확정
3. Supabase Redirect URL 설정
4. Google 로그인 연동
5. 실제 사용자 테스트
6. 반복 사용 지표를 보고 앱스토어 확장 검토
