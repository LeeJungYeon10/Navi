# 안녕나비야 - 개발 컨텍스트 (Cursor용)

> 이 파일은 Cursor에서 개발할 때 항상 참조할 수 있도록 만든 마스터 컨텍스트 파일이다.

---

## 1. 프로젝트 개요

**프로젝트명**: 안녕나비야 (Hello Naviya)  
**컨셉**: 따뜻한 반려묘 AI와 대화하며 자연스럽게 건강 데이터를 수집하고, 부드럽게 healthier habits를 제안하는 앱  
**타겟**: 바쁜 현대인 (특히 20~40대)  
**MVP 목표**: 게스트 모드 → 로그인 유도 → 자연스러운 대화 기반 데이터 수집 + 행동 유도 루프 완성

**핵심 차별점**:
- 기록/설문이 아닌 **대화**로 데이터 수집 (Stealth Assessment)
- "반려묘"라는 감정적 유대감 중심
- 사용자에게 부담을 주지 않는 부드러운 피드백

---

## 2. 핵심 기술 결정 사항 (MVP)

| 항목 | 결정 내용 |
|------|----------|
| **대화 생성 방식** | 구조화된 시스템 프롬프트 + 압축 상태 주입 |
| **상태 주입 형식** | `level:3 \| mood:피곤 \| bond:64 \| recent:수면부족` 형태로 최소화 |
| **데이터 수집** | Stealth Assessment (한 번의 API 호출로 응답 + 분석 동시 처리) |
| **Stealth Assessment 출력** | JSON 구조로 `reply_message` + `assessment` 반환 |
| **비적절 대화 대응** | "나비는 고양이라 그런 말은 잘 모르겠어..." 스타일로 부드럽게 선 긋기 |
| **사용자 피드백 스타일** | 사실 기반 + 부드러운 격려 (예: 수면시간 변화 언급 + "앞으로도 관리해봐요") |
| **하드코딩** | 최소화. 구조화된 프롬프트로 제어 |

---

## 3. 중요한 규칙 (절대 위반 금지)

### 응답 톤 & 스타일
- 사용자에게 보여주는 피드백은 **사실 기반 + 부드러운 격려** 스타일로 작성
- "네겐트로피"는 초기 기획에서 생활 리듬이 정리되고 회복되는 방향을 설명하기 위해 쓰인 **내부 용어**로만 유지
- "질서를 찾았다", "네겐트로피", "내면의 질서" 같은 철학적/추상적 표현은 사용자에게 보이는 문구에 **절대 사용 금지**
- 사용자에게는 "생활 리듬", "오늘의 상태", "작은 회복 신호", "좋은 변화"처럼 쉽고 구체적인 표현 사용
- 명령형/지시형 표현 피하기 ("~해라", "~해야 해", "~해줘")

### 비적절한 대화 대응
- 성적 대화, 욕설, 폭언, 공격적 언어에 대해서는 **"나비는 고양이라 그런 말은 잘 모르겠어..."** 스타일로 부드럽지만 단호하게 대응
- 사용자를 공격하거나 비난하지 않으면서 선을 긋는 태도 유지

### Stealth Assessment
- 사용자가 직접 말하지 않은 내용은 추측하지 말 것
- 분석 결과는 사용자에게 노출하지 않고 백엔드에만 저장
