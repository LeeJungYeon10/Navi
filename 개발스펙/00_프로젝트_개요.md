# 🦋 안녕나비야 (Hello Naviya) - 개발 스펙 종합

**프로젝트명**: 안녕나비야  
**컨셉**: 매일 대화하는 온라인 반려묘 AI (감정적 유대감 + 건강 증강 코칭)  
**타겟**: 바쁜 현대인, 특히 정신건강 + 신체건강을 동시에 관리하고 싶은 20~40대  
**MVP 목표**: 게스트 모드 → 로그인 유도 → 대화 기반 데이터 수집 + 행동 유도 + 시각적 보상 루프 완성

---

## 1. 프로젝트 궁극적 목표

> 겉으로는 다정하고 따뜻한 반려묘와의 대화 형태를 띠지만, 이면에는 사용자의 수면, 활동, 감정 등의 일상 데이터를 자연스럽게 파악해 부드럽게 healthier habits를 제안하는 **대화형 건강 코칭 시스템**을 구축한다.

### 핵심 지향점
- 사용자 건강리터러시 개선
- 사용자 바이오리듬 개선
- 생애주기별 맞춤형 피드백
- 자연스러운 대화를 통한 유대감 형성
- 정서적 지지 제공

---

## 2. 서비스 컨셉 (포지셔닝)

**“온라인 반려묘”**

- 기록이나 차트가 아닌, **대화를 통해 자연스럽게 데이터 수집**
- 단순한 조언이 아닌, **정서적 지지 + 실질적 건강 피드백** 동시 제공
- “기록하는 앱”이 아니라 **“함께 살아가는 존재”**로 느껴지게 함

---

## 3. 핵심 차별점

| 기존 건강 앱 | 안녕나비야 |
|--------------|------------|
| 데이터 입력 부담 | 대화 속 자연스러운 수집 (Stealth Assessment) |
| 정신/신체 분리 관리 | 하나의 대화로 통합 관리 |
| 지시형 조언 | 반려묘 스타일의 부드러운 제안 + 보상 |
| 기록 중심 | 관계(유대감) + 성장 중심 |

---

## 4. MVP 범위 (우선순위)

### Phase 0 (MVP 핵심)
- 게스트 모드 즉시 대화 시작
- AI 대화 + 은밀한 감정/상태 수집
- 건강 증강 행동 제안 + 완료 피드백 루프
- 로그인 유도 (대화 4턴 이후)
- 게스트 데이터 → 로그인 계정 마이그레이션
- 나비 정원 (타임라인 + 간단한 기록 보기)

### Phase 1 (고도화)
- 유대감 / 성장 시스템 본격 적용
- 나비 상태 (에너지, 기분, 스트레스) 시각화
- 더 정교한 개인화 코칭

---

## 5. 기술 스택 제안 (개발 시작 시 참고)

- **Frontend**: Next.js 14 (App Router) or React Native (모바일 우선 고려)
- **Backend**: Supabase (PostgreSQL + Auth + Storage) 또는 Firebase
- **AI**: OpenAI GPT-4o 또는 Claude 3.5 (프롬프트 엔지니어링 중요)
- **상태 관리**: Zustand 또는 Jotai
- **디자인**: Tailwind + Framer Motion (미세 애니메이션)
- **인증**: Google OAuth (Supabase Auth 추천)

## 7. 대화 시스템 아키텍처 결정 사항 (MVP)

**의사결정 배경**:
- 완전 하드코딩은 확장성과 유지보수 측면에서 비효율적
- 매 턴마다 전체 상태를 주입하는 것은 토큰 비용과 복잡도가 높음
- MVP 단계에서는 **제어와 일관성 확보**를 우선으로 하되, 과도한 복잡도는 피하는 방향으로 결정

**MVP 적용 방식**:
- **구조화된 시스템 프롬프트 + 압축 상태 주입** 하이브리드 방식 채택
- 나비의 기본 페르소나, 말투 규칙, 금지사항은 시스템 프롬프트에 고정
- 매 턴에는 **압축된 상태**만 주입 (예: `level:3 | mood:피곤 | bond:64 | recent:수면 부족`)
- 완전 하드코딩은 피하고, LLM이 상태를 참고해 자연스럽게 생성하도록 유도
- MVP에서는 제어 가능성과 응답 일관성을 어느 정도 확보하는 데 집중

**장기 목표 (MVP 이후)**:
- Phase 1 이후: Prompt Caching + 대화 요약(RAG) 도입으로 토큰 효율화 및 빠른 반복 개선
- Phase 2: Memory Layer 분리 및 더 정교한 개인화

이 방식으로 MVP에서는 응답의 제어와 일관성을 확보하면서도, 이후 단계에서 빠르게 개선할 수 있는 기반을 마련할 수 있을 것으로 기대함.
