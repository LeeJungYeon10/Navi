# 안녕나비야 - 데이터 모델 (ERD / Schema)

## 설계 철학
1. 최소한의 개인정보 수집 (Google 로그인 기반)
2. 대화 맥락과 건강 데이터의 결합
3. 피드백 루프(행동 이행 여부) 측정 및 시각화 가능
4. 추후 Neuralink / 웨어러블 연동을 고려한 확장성 확보

---

## 1. 핵심 테이블 (MVP)

### ① Users (사용자 관리)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| user_id | PK (String) | Google 연동 고유 식별자 |
| email | String | Google 이메일 |
| nickname | String | 사용자가 설정한 애칭 (나비가 부를 이름) |
| segment_type | String | 대화로 수집된 생활 패턴 세그먼트 (예: Night_Owl, Early_Bird) |
| created_at | DateTime | 가입일 |

### ② Chats (대화 로그)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| chat_id | PK (UUID) | 메시지 고유 번호 |
| user_id | FK | User 테이블 연결 |
| sender_type | Enum | 'USER' 또는 'AI' |
| message | Text | 대화 내용 |
| timestamp | DateTime | 대화 발생 시각 |

### ③ Emotion_Logs (감정 및 상태 분석) - Stealth Assessment
| 컬럼 | 타입 | 설명 |
|------|------|------|
| log_id | PK (UUID) | 로그 고유 ID |
| chat_id | FK | 특정 대화와 1:1 매핑 |
| user_id | FK | 사용자 연결 |
| emotion_tag | String | 분석된 감정 키워드 (예: 우울, 불안, 평온, 피로, 행복, 기쁨, 즐거움, 안정) |
| physical_state_tag | String | 분석된 신체 상태 (예: 수면부족, 피로도높음, 눈의피로) |
| created_at | DateTime | 분석 시각 |

### ④ Action_Logs (피드백 루프 및 행동 기록)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| action_id | PK (UUID) | 행동 로그 ID |
| user_id | FK | 사용자 연결 |
| action_type | Enum | 'SPATIAL' (환기/조명), 'PHYSICAL' (산책/스트레칭), 'BIO_RHYTHM' (물마시기/심호흡) |
| is_completed | Boolean | 행동 이행 여부 (True: 완료) |
| completed_at | DateTime | 행동 완료 시각 |
| reward_given | Boolean | 보상 지급 여부 |

---

## 2. 테이블 간 관계

- **Users : Chats** → 1 : N
- **Chats : Emotion_Logs** → 1 : 1 (하나의 대화 = 하나의 감정 분석)
- **Users : Action_Logs** → 1 : N (하루에 여러 행동 수행 가능)
