# 안녕나비야 - LLM 기반 실시간 감정 분석 가이드

> Stealth Assessment의 핵심 기술인 **LLM 기반 실시간 감정/상태 분석**에 대한 설계와 구현 가이드.

---

## 1. 왜 LLM으로 실시간 감정 분석을 하는가?

| 방식 | 장점 | 단점 | 안녕나비야 적합도 |
|------|------|------|------------------|
| 규칙 기반 / 키워드 매칭 | 빠르고 저렴함 | 맥락 이해 부족, 정확도 낮음 | 낮음 |
| 전용 감정 분석 모델 (BERT 등) | 정확도 높음 | 유지보수 비용 큼, 실시간 연동 복잡 | 보통 |
| **LLM 기반 분석** | 맥락 이해 우수, 유연함 | 토큰 비용 + 지연 발생 가능 | **높음** |

**선택 이유**:
- 단순 키워드가 아닌 **대화 전체 맥락**을 이해해야 함
- Stealth Assessment처럼 **자연스럽게** 데이터를 수집해야 함
- 별도 모델 운영 없이 높은 품질을 기대할 수 있음

---

## 2. 실시간 감정 분석 구현 방식 (MVP 추천)

**가장 현실적이고 효율적인 방법: 한 번의 API 호출로 처리**

### 흐름
```
사용자 메시지 수신
        ↓
[시스템 프롬프트 + 압축 상태 + 분석 지시]
        ↓
LLM 호출 (1회)
        ↓
{
  "reply_message": "...",
  "assessment": { ... }
}
```

**장점**:
- Latency 최소화
- API 비용 절감
- 응답 생성과 분석을 동시에 처리

---

## 3. 정확도를 높이기 위한 프롬프트 기법

### 1. 명확한 지시 포함
```text
너는 사용자의 대화에서 감정과 신체 상태를 분석하는 역할을 해.
반드시 아래 규칙을 지켜:
- 사용자가 직접 언급한 내용만 기반으로 분석할 것
- emotion_tags와 physical_state_tags는 지정된 태그 중에서 선택
- intensity는 low / medium / high 중 하나로 판단
```

### 2. Few-shot 예시 제공 (강력 추천)
프롬프트에 3~5개의 구체적인 예시를 넣으면 분석 정확도가 크게 향상된다.

### 3. 출력 형식을 JSON으로 강제
```text
반드시 아래 JSON 형식으로만 출력해:
{
  "reply_message": "...",
  "assessment": {
    "emotion_tags": [...],
    "physical_state_tags": [...],
    ...
  }
}
```

### 4. 태그 후보 제한
무한정 태그를 생성하지 않도록, 가능한 태그 목록을 프롬프트에 미리 명시하는 것이 좋다.

---

## 4. 현재 우리가 설계한 구조 요약

| 항목 | 내용 |
|------|------|
| **분석 주체** | LLM (GPT-4o 또는 Claude 3.5) |
| **분석 타이밍** | 사용자 메시지 수신 시 (실시간) |
| **출력 방식** | 응답 + 분석 결과를 하나의 JSON으로 반환 |
| **저장 위치** | `Emotion_Logs` 테이블 |
| **다음 턴 활용** | 압축 상태(`mood`, `recent` 등)로 AI 프롬프트에 주입 |
| **비용 최적화** | 압축 상태 주입 + 한 번의 API 호출 |

---

## 5. 실무에서 고려할 점

| 이슈 | 해결 방향 | 비고 |
|------|-----------|------|
| **지연 시간** | Structured Output / JSON mode 사용 | Claude의 JSON mode가 특히 강력 |
| **비용** | 간단한 대화는 분석 생략하거나 간소화 | 규칙 기반 필터링과 병행 가능 |
| **정확도** | Few-shot 예시 + 태그 후보 제한 | 가장 효과적인 방법 |
| **프라이버시** | 분석 결과는 사용자에게 노출하지 않음 | Stealth Assessment의 핵심 |
