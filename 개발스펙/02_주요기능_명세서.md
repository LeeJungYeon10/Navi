# 🦋 안녕나비야 - 주요 기능 명세서 (Functional Specification)

> **작성 기준**: 대화 몰입도를 극대화하고, 로컬 스토리지와 서버 DB 간 데이터 마이그레이션을 안정적으로 처리하는 데 중점.

## 기능 목록 (MVP)

| 기능 ID | 카테고리 | 기능명 | 기능 상세 로직 | 연동 데이터 / API |
|---------|----------|--------|----------------|-------------------|
| **F-01** | 온보딩 | 게스트 세션 생성 | 앱 최초 접속 시, 별도의 가입 절차 없이 브라우저 로컬 스토리지에 임시 `guest_id` 생성하여 대화 기록 저장 | Local Storage |
| **F-02** | 온보딩 | 오프닝 튜토리얼 | 첫 화면 진입 시 "오늘 나는 널 만나서 기분이 좋아. 너의 기분은 어때?" 텍스트가 타이핑 애니메이션(1.5초)으로 출력. 첫 메시지 전송 후 하단 툴팁 자동 소멸 | Front-end UI |
| **F-03** | 대화 (Home) | AI 채팅 송수신 | 사용자의 텍스트 입력을 받아 AI 모델로 전송하고, 반환된 답변을 말풍선 UI로 렌더링. 대화가 길어질 경우 자동 스크롤 처리. **MVP는 구조화된 시스템 프롬프트 + 압축 상태 주입 방식**으로 구현 (친밀도/상태에 따른 응답 제어 우선) | OpenAI / Claude API |
| **F-04** | 대화 (Home) | 은밀한 데이터 수집 (Stealth Assessment) | AI 서버에서 사용자의 대화 텍스트를 분석하여 핵심 감정과 생체 상태를 추출해 메타데이터로 태깅. 분석 결과는 압축 상태로 다음 턴 프롬프트에 반영 | AI Prompt Engineering + DB (Emotion_Log) |
| **F-05** | 대화 (Home) | 행동 유도형 퀵 리플라이 | AI가 특정 행동(환기, 물 마시기 등)이나 질문을 던질 때, 텍스트 입력창 상단에 선택형 버튼(Quick Reply) 생성. 버튼 클릭 시 해당 텍스트가 사용자 말풍선으로 자동 전송 | Front-end UI |
| **F-06** | 대화 (Home) | 피드백 루프 및 시각적 보상 | 사용자가 제안된 행동을 완료(퀵 리플라이 클릭)하면, AI가 사실 기반으로 부드럽게 피드백 (예: "수면시간이 늘었네요. 6.4~7.8시간 정도 유지하는 게 건강한 일상에 도움이 돼요. 앞으로도 주기적으로 관리해봐요") + 배경의 미세한 시각적 변화 1.5초간 노출 | Front-end Animation + DB (Action_Log) |
| **F-07** | 대화 (Home) | 로그인 유도 트리거 | 게스트 세션에서 대화 턴(Turn)이 4회에 도달하면, 다음 AI 답변 시 [Google로 시작하기] 버튼을 포함한 초대 말풍선 또는 하단 팝업 노출 | Front-end UI |
| **F-08** | 인증 | Google OAuth 로그인 | [Google로 시작하기] 클릭 시 구글 로그인 페이지로 리다이렉트. 인증 성공 시 고유 식별자(UID) 및 이메일 정보를 받아와 서버 세션 생성 | Google OAuth 2.0 API |
| **F-09** | 인증 | 데이터 마이그레이션 | 구글 로그인 성공 직후, 로컬 스토리지에 있던 게스트 대화 내역(F-01)을 서버 DB의 해당 유저 계정으로 이관(Upload)하고 로컬 데이터는 삭제 | REST API (POST) |
| **F-10** | 정원 (History) | 타임라인 렌더링 | 로그인 유저에 한해 진입 가능. 서버 DB에서 해당 유저의 과거 대화 날짜, 추출된 감정 태그, 수행한 건강 증강 행동 내역을 불러와 리스트/캘린더 형태로 시각화 | REST API (GET) |
| **F-11** | 설정 (My Page) | 회원 탈퇴 처리 | '탈퇴하기' 클릭 시, 해당 유저의 모든 대화 기록, 감정 메타데이터, 개인정보(이메일)를 DB에서 영구 삭제 (Hard Delete) | REST API (DELETE) |

## 개발자 소통용 보충 노트

- **F-03 / F-04 (대화 시스템)**: MVP에서는 **완전 하드코딩 대신 구조화된 프롬프트 + 압축 상태 주입** 방식으로 구현. 매 턴에 전체 상태를 다 넣는 대신 `level:3 | mood:피곤 | bond:64` 형태로 압축하여 토큰 비용과 복잡도를 낮춤. 제어 가능성과 응답 일관성을 우선으로 확보하는 방향 (빠른 반복 개선은 MVP 이후 목표).
- **F-04 (데이터 수집)**: AI 응답 속도 최적화를 위해, 사용자에게 보여줄 '채팅 답변 텍스트'와 백엔드에 저장할 '감정 분석 JSON 데이터'를 **한 번의 API 호출**로 동시에 받아오도록 AI 프롬프트를 설계해야 함.
- **F-06 (시각적 보상)**: 시각적 요소의 변화는 사용자의 회복 과정을 은유적으로 표현. 디자인 에셋 준비 전까지는 CSS 속성(Opacity, Color Transition)만으로 가볍게 구현 가능.
