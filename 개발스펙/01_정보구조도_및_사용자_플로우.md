  function setSpecLang(lang) {
    document.querySelectorAll('[data-spec-lang]').forEach((el) => {
      el.hidden = el.dataset.specLang !== lang;
    });
    document.getElementById('specLangKo')?.classList.toggle('is-active', lang === 'ko');

    try { localStorage.setItem('navi-spec-ia-lang', lang); } catch (_) {}
  }
  (function initSpecLang() {
    var saved = 'ko';
    try { saved = localStorage.getItem('navi-spec-ia-lang') || 'ko'; } catch (_) {}
    setSpecLang(saved === 'en' ? 'en' : 'ko');
  })();
</script>

# 안녕나비야 - 정보구조도 (IA) 및 사용자 플로우

## 1. 전체 앱 구조 (MVP)

```
안녕나비야
├── 1. 스플래시 / 온보딩 (Guest Mode)
│   └── 즉시 대화 시작 (로그인 장벽 없음)
│
├── 2. 메인 홈 - 나비와의 대화 (Chat Interface)
│   ├── 오프닝 튜토리얼
│   ├── 채팅 스트림
│   ├── 퀵 리플라이 (Quick Reply)
│   ├── 행동 완료 피드백 + 시각적 보상
│   └── 로그인 유도 트리거 (대화 4턴 후)
│
├── 3. 나비 정원 (Garden - History & Insights)
│   └── 로그인 유저 전용
│       ├── 타임라인 (날짜별 대화 + 감정 태그)
│       └── 생활 리듬 / 행동 기록 요약
│
└── 4. 마이페이지 (My Page)
    ├── 프로필 (닉네임 수정)
    ├── 설정 (푸시, 테마, 언어)
    └── 계정 관리 (로그아웃, 탈퇴)
```

---

## 2. 사용자 플로우 상세

### Flow 1: 게스트 사용자 (최초 진입)
1. 앱 실행 → 스플래시
2. 즉시 나비와 대화 시작 (타이핑 애니메이션)
3. 자연스러운 대화 진행
4. 4턴 정도 대화 후 → "대화를 보관할까요?" 로그인 유도
5. Google 로그인 → 게스트 데이터 자동 마이그레이션

### Flow 2: 로그인 사용자
1. 앱 실행 → 마지막 대화 상태 복원
2. 대화 계속
3. 건강 증강 행동 제안 → 완료 시 즉시 보상
4. 나비 정원에서 자신의 기록 확인
5. 마이페이지에서 설정 관리

### Flow 3: 언어 전환 (한국어 ↔ English)
1. 마이페이지 → 설정 → 언어 선택 (한국어 / English)
2. UI 문구, 퀵 리플라이, 나비 응답 언어 즉시 전환
3. 선택 언어는 Local Storage에 저장 (게스트·로그인 공통)
4. 로그인 후에도 동일 언어 설정 유지

---

## 3. 핵심 화면 목록 (MVP)

| 화면 ID | 화면 이름 | 접근 조건 | 주요 기능 |
|---------|-----------|-----------|-----------|
| S-01 | 스플래시 | - | 나비 그래픽 + 로고 |
| S-02 | 메인 대화 (Home) | 게스트/로그인 | 채팅, 퀵 리플라이, 행동 피드백 |
| S-03 | 로그인 유도 바텀시트 | 대화 4턴 후 | Google 로그인 유도 |
| S-04 | 나비 정원 | 로그인 필수 | 타임라인, 기록 요약 |
| S-05 | 마이페이지 | 로그인 필수 | 프로필, 설정(푸시·테마·언어), 탈퇴 |

---

## 4. 데이터 흐름 개요

- **게스트 기간**: Local Storage에 대화 임시 저장
- **로그인 순간**: Local Storage 데이터 → 서버 DB로 마이그레이션
- **로그인 후**: 모든 데이터 서버 동기화
- **감정/상태 데이터**: 대화 중 AI가 백엔드에 은밀히 태깅 (Stealth Assessment)
- **언어 설정**: Local Storage `language` 키에 `ko` / `en` 저장, UI·응답에 즉시 반영

# Hello Naviya - Information Architecture (IA) & User Flow

## 1. Overall App Structure (MVP)

```
Hello Naviya
├── 1. Splash / Onboarding (Guest Mode)
│   └── Start chatting immediately (no login barrier)
│
├── 2. Main Home - Chat with Navi (Chat Interface)
│   ├── Opening tutorial
│   ├── Chat stream
│   ├── Quick Reply
│   ├── Action completion feedback + visual reward
│   └── Login prompt trigger (after 4 chat turns)
│
├── 3. Navi Garden (History & Insights)
│   └── Logged-in users only
│       ├── Timeline (daily chats + emotion tags)
│       └── Life rhythm / action record summary
│
└── 4. My Page
    ├── Profile (nickname edit)
    ├── Settings (push, theme, language)
    └── Account management (logout, delete account)
```

---

## 2. Detailed User Flows

### Flow 1: Guest User (First Visit)
1. Launch app → splash screen
2. Start chatting with Navi immediately (typing animation)
3. Continue natural conversation
4. After ~4 turns → login prompt: "Would you like to save this conversation?"
5. Google sign-in → automatic guest data migration

### Flow 2: Logged-in User
1. Launch app → restore last conversation state
2. Continue chatting
3. Health-boost action suggestions → instant reward on completion
4. Review personal records in Navi Garden
5. Manage settings in My Page

### Flow 3: Language Switch (Korean ↔ English)
1. My Page → Settings → Language (Korean / English)
2. UI copy, quick replies, and Navi reply language switch immediately
3. Selected language is saved in Local Storage (guest and logged-in)
4. Language preference persists after login

---

## 3. Core Screen List (MVP)

| Screen ID | Screen Name | Access | Key Features |
|-----------|-------------|--------|--------------|
| S-01 | Splash | - | Navi graphic + logo |
| S-02 | Main Chat (Home) | Guest / logged-in | Chat, quick reply, action feedback |
| S-03 | Login Prompt Bottom Sheet | After 4 turns | Google sign-in prompt |
| S-04 | Navi Garden | Login required | Timeline, record summary |
| S-05 | My Page | Login required | Profile, settings (push·theme·language), delete account |

---

## 4. Data Flow Overview

- **Guest period**: Conversations temporarily stored in Local Storage
- **At login**: Local Storage data → migrated to server DB
- **After login**: All data synced to server
- **Emotion/state data**: AI silently tags backend during chat (Stealth Assessment)
- **Language setting**: Stored in Local Storage as `language` (`ko` / `en`), applied immediately to UI and replies
